 MILESTONE 2 — GNOME visual identity: dots + capsule                     [x] DONE
-------------------------------------------------------------------------------
Goal: the signature look — dim circles + a wider highlighted "pill" on the current desktop
that moves between positions. Horizontal panel first. (ref: kirigami.md, qml-performance.md)
NOTE: M2 first shipped a separate SLIDING OVERLAY pill; that was replaced after M3 by a GNOME-
accurate REFLOW + morph model (each element morphs dot⇄capsule; uniform spacing) — see CLAUDE.md
"Visual model". The bullets below reflect the current model.
  [x] Inactive dot: Rectangle radius=h/2, Kirigami.Theme.textColor @ ~0.45 opacity
  [x] Active capsule: each WorkspaceDot IS the element and morphs to a wider rounded capsule
      when active (no overlay). ONE uniform Row spacing (dotSize*spacingFactor) sits between
      every pair, so pill-to-dot gap == dot-to-dot gap (GNOME look); the active element widens
      in place and the Row reflows, so it can never cover/clip a neighbour (no overhang math).
  [x] Morph on switch: Behavior on width (OutCubic) + colour + opacity, durations from
      Kirigami.Units (guard 0 when animations disabled). `animate` latch (Qt.callLater) →
      instant first placement (active is already a capsule on reload), morph later switches.
  [x] Active colour = Kirigami.Theme.highlightColor @ full opacity (theme-following by default)
  [x] Size everything via Kirigami.Units (HiDPI); no hardcoded px (pillWidthFactor 3.5,
      spacingFactor 0.5, inactiveOpacity 0.45, hoverOpacity 0.8 are dimensionless, M5-configurable);
      implicitWidth advertised via a stable formula (one capsule + dots) so the panel cell doesn't jitter.
VERIFY: matches GNOME feel; capsule morphs smoothly on switch; uniform tight spacing; follows
        color scheme; looks correct on HiDPI / fractional scaling.
        DONE: make check (unit+integration incl. uniform-spacing/exactly-one-capsule/morph/
        sizing) + make lint clean. In-shell smoke test (make dev/restart) is the user's manual step.

 MILESTONE 4 — Form factor & sizing robustness                          [x] DONE
-------------------------------------------------------------------------------
Goal: correct in every placement. (ref: plasmoid.md)
NOTE: orientation flows one way — main.qml reads Plasmoid.formFactor and passes a plain
`vertical` bool down to WorkspaceIndicator -> WorkspaceDot (keeps the sub-components free of
Plasmoid/PlasmaCore and headless-testable). Layout is TWO nested Grid positioners: the outer stacks
the grid LINES along the cross axis, the inner lays each line's dots along the major axis; only the
line dimension is pinned to 1, the other is -1/auto so it tracks the live child count and never
warns on add/remove. The 1/-1 pair flips with `vertical` (no Row/Column type swap).
  [x] Horizontal panel -> rows of dots; vertical panel -> columns of dots
      (switch on Plasmoid.formFactor via the `vertical` bool)
  [x] Capsule morphs along the correct axis (widens horizontally, grows tall vertically);
      radius pinned to dotSize/2 so the ends stay stadium-round both ways; a Behavior per axis
  [x] Correct implicit sizing in both orientations: the MAJOR (line) axis is pinned (Layout
      min=pref=max = stripLength) and the CROSS axis carries the lines (min/pref = crossThickness,
      max -1 -> unconstrained) so the panel fills the thickness; one formula drives both axes
  [x] Multi-row grid mirrors KWin: read VirtualDesktopInfo.desktopLayoutRows (reactive, public,
      null-guarded) and split the desktops into that many independent reflow LINES — each keeps the
      exact single-line dots+pill look (lines centred independently, not column-aligned). Mirrors
      System Settings -> Virtual Desktops -> "Rows" live; no widget setting. Default 1 = single line.
      perLine = Logic.gridColumns(count, rows) = ceil(count/rows); lines = Logic.chunk(ids, perLine).
  [x] Sensible behavior on the desktop containment (Planar) and when floating — both report
      non-Vertical, so they fall through to the horizontal row (the off-panel default)
  [x] Set Plasmoid.status (PassiveStatus) and backgroundHints (NoBackground — dots float on the
      bare panel, the GNOME look; making it user-configurable is deferred to M5)
VERIFY: move widget between top/bottom panel, a side panel, and the desktop — layout
        and sizing stay correct.
        DONE: make check (129 tests: existing + new vertical and multi-row unit/integration —
        vertical morph/radius/stacking/sizing, gridColumns/chunk logic, KWin-rows mirroring,
        uneven last line, reactive row changes, 2-D sizing, second-line capsule, vertical
        transpose) + make lint clean. In-shell smoke test (make dev/restart, then move the widget
        between a horizontal panel, a vertical side panel, and the desktop, and raise System

 MILESTONE 6 — Robustness hardening & edge cases                        [x] DONE
-------------------------------------------------------------------------------
Goal: never break — the whole reason for this project. (ref: robustness.md)
  [x] Plasma 6.7 "switch desktops independently for each screen" (PerOutputVirtualDesktops):
      each pager shows ITS monitor's current desktop, not the global one. WorkspaceIndicator reads
      Screen.name (QtQuick) and vdi.currentDesktopByScreenName(name) (public, org.kde.taskmanager),
      recomputed on currentDesktopForScreenChanged via Logic.resolveCurrentDesktop (per-screen else
      global; typeof-guarded for older Plasma). No new setting (auto-mirrors KWin). Write stays the
      global switch (no public per-output write; KWin routes it to the active output). Covered by
      tst_logic + tst_workspaceindicator; read API verified live (DP-6 != DP-5). (ref: virtual-desktops.md)
  [x] Audit imports: public only, versionless, zero org.kde.plasma.private.*. Audited every .qml/.js
      (production + config pages + tests): no private.* anywhere, no versioned imports. The only
      off-allowlist production import is org.kde.kquickcontrols (ColorButton) in the lazy-loaded
      ConfigAppearance page — the documented, sanctioned exception (CLAUDE.md): a config page is never
      instantiated by the always-on widget, so a break there cannot kill the running pager.
  [x] Guard all desktopIds/currentDesktop access against transient empty/null. Already funnelled
      through null-safe views (desktopIds/desktopNames `?? []`, currentDesktop via indexOf → -1) and
      clamped logic.js helpers; the one remaining bare scalar read (vdi.numberOfDesktops in addDesktop)
      now uses `?? 0`. New integration test asserts an empty desktopIds ARRAY (distinct from a null
      source) yields zero dots and a one-dot cell that never collapses to 0.
  [x] Edge cases: exactly 1 desktop — Logic.canRemoveDesktop blocks removing the last on every path;
      the "Remove Last Desktop" action is disabled/greyed at count 1 (kept visible — standard Plasma
      pattern). Many desktops — SCALE-TO-FIT: when the natural strip would exceed the panel-allocated
      length, the dots/pill shrink to fill it (Logic.fitDotSize, the inverse of lineExtent) instead of
      overflowing onto the neighbours, capped at the natural size and floored at a legible minimum; the
      major-axis Layout MINIMUM drops to that floor so the panel can compress us (preferred/maximum stay
      natural, so the common case is byte-for-byte unchanged and there is no binding loop — the hints
      depend only on the NATURAL size). Rapid repeated switching — the one-way `animate` latch +
      idempotent updateCurrentDesktop() converge deterministically; new test storms 12 back-to-back
      switches and asserts exactly one capsule on the final target.
  [x] Respect "reduce animations" (Kirigami.Units.longDuration === 0): centralised in
      Logic.effectiveDuration (0 always wins over any configured duration) → WorkspaceDot.morphEnabled,
      which gates all four morph Behaviors → instant. Exhaustively covered at the logic tier.
  [x] qmllint clean (treat warnings as errors): only the documented benign i18n/i18np unqualified
      warnings remain (a plasmoid global) — zero real defects, zero unresolved private/renamed symbols.
      qmlformat: the repo uses a deliberate compact hand-formatted style (one-line data rows, inline
      object literals) that qmlformat's default would expand everywhere; the new code matches the
      surrounding style, so applying `qmlformat -i` would only add noise and was intentionally not run
      (qmllint is the safety net per robustness.md).
  [ ] Re-test after a real plasmashell restart and (if possible) a Plasma point update — USER's manual
      step (see DONE note); cannot be automated headless.
VERIFY: no QML errors/warnings in journalctl; survives shell reload; no crashes on
        desktop add/remove storms.
        DONE: make check (210 tests: +Logic fitDotSize fit/unbounded, +WorkspaceIndicator scale-to-fit
        shrink/ample/vertical, empty-array zero-dots, many-desktops single+grid, rapid-switch convergence;
        the two Layout-sizing tests updated for the floor minimum) + make lint clean. In-shell smoke test
        (make dev/restart, then: add desktops until the strip would exceed a short/crowded panel and
        confirm the dots SHRINK to fit rather than overflow the neighbours, remove them and confirm they
        grow back; toggle System Settings → Virtual Desktops → "Rows"; switch desktops rapidly; enable
        "reduce animations" for instant morphs) — and a re-test after the next Plasma point update — are
